Examen Git y Github

Alumn@: LAS

  1. ¿Que es un sistema de control de versiones?

    es una herramienta que ayuda a los desarrolladores de software a gestionar los cambios en el codigo fuente a lo largo del tiempo y permite guardar instantaneas del codigo en diferentes puntos del tiempo lo que facilita trabajar con él

  2. ¿Cual es la diferencia entre git y github?

    Git es un programa que se ejecuta en tu computadora para gestionar los cambios en los archivos de un proyecto. Es como un "controlador" local.

    GitHub es una plataforma online que usa Git. Es como un sitio web donde puedes guardar y compartir tus proyectos con otros.

    GitHub también te ayuda a trabajar en equipo, haciendo cosas como "pull requests" para que revisen tu código antes de fusionarlo con el principal.

  3. ¿Cómo creo una rama en git?

    Para crear una rama en Git, simplemente usas este comando: git branch nombre_de_la_rama

    Y para cambiarte a esa rama, usas: git checkout nombre_de_la_rama

    Si quieres hacer ambas cosas (crear y cambiar) en un solo paso: git checkout -b nombre_de_la_rama

  4. ¿Cómo clono un repositorio?

    Cuando alguien ya tiene un repositorio en GitHub, lo puedes "clonar" para tener una copia local en tu computadora.

    El comando es: git clone URL_del_repositorio

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Para traer los cambios más recientes que alguien haya subido al repositorio (si no estás trabajando en ellos),

    el comando es: git pull

    Este comando descarga los cambios y los mete en tu rama actual.

  6. ¿Cómo puedo gener un tag?

    Un tag es como una "etiqueta" que marcas en un commit específico.

    El comando es: git tag nombre_del_tag

    Si quieres agregar una pequeña descripción (por ejemplo, para una versión),

    usas: git tag -a nombre_del_tag -m "Versión 1.0"

  7. Cómo hago una mezcla de ramas?

    Cuando quieres juntar los cambios de una rama con otra,

    usas: git merge nombre_de_la_rama

    Primero tienes que estar en la rama a la que quieres fusionar los cambios. Por ejemplo, si estás en master y quieres fusionar la rama desarrollo,

    solo haces: git checkout master, git merge desarrollo

  8. ¿Cómo puedo moverme a un commit especifico?

    Si quieres ir a un commit anterior para ver cómo estaba el código en ese momento,

    usas el hash del commit:git checkout hash_del_commit

  9. ¿Cómo puedo borrar una rama?

    Si ya no necesitas una rama,

    puedes borrarla con: git branch -d nombre_de_la_rama

    Si la rama no ha sido fusionada, y aún así quieres borrarla,

    usas: git branch -D nombre_de_la_rama

    Para borrar una rama remota: git push origin --delete nombre_de_la_rama

  10. ¿Cómo puedo borrar un tag?

    Para borrar un tag localmente: git tag -d nombre_del_tag

    Y si quieres borrarlo en el repositorio remoto: git push origin --delete nombre_del_tag

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Para ver qué archivos has cambiado y si ya los has preparado para enviarlos (staged),

    usas: git status

  12. ¿Para que sirve el comando git remote -v?

    Te muestra las URLs de los repositorios remotos, es decir, de dónde estás descargando los cambios y a dónde los vas a subir

    el comando es: git remote -v

  13. ¿Cómo le digo a git cual es mi email y correo?

    Puedes configurar tu nombre y correo con estos comandos:

    git config --global user.name "Tu Nombre"
    git config --global user.email "tuemail@dominio.com"

  14. ¿Cuales son los pasos para conectar git con github?

    1.Genera una clave SSH (para evitar poner tu usuario y contraseña cada vez que empujes cambios):

    ssh-keygen -t rsa -b 4096 -C "tuemail@dominio.com"

    2.Agrega la clave SSH al agente:

    eval "$(ssh-agent -s)"
    ssh-add ~/.ssh/id_rsa

    3.Copia la clave pública:

    cat ~/.ssh/id_rsa.pub

    4.Pega la clave SSH en GitHub: Ve a GitHub -> Configuración -> SSH and GPG keys -> Nueva clave SSH.

    5.Clona un repositorio usando SSH:

    git clone git@github.com:usuario/repositorio.git
